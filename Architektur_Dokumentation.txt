1. SYSTEMÜBERBLICK (KURZ)
================================================================================

1.1 Architektur-Typ
-------------------
  Multi-Page-Website (MPA) — ~25 HTML-Seiten im Repo-Root
  + Node/Express-Server unter server/ (TTS-API, OmniQR-API, i18n-API, Ops)
  + statische Auslieferung des gesamten SITE_ROOT via express.static

1.2 Datenfluss (vereinfacht)
----------------------------
  Browser → HTML/CSS/JS (Root) → i18next lädt assets/locales/*.json
  Vorlesen  → POST /api/speak → server/tts-router.js → XTTS | ElevenLabs | OpenAI
  OmniQR    → /omniqr-ai-for-tourist-of-thailand/api/* → server/omniqr-…/router.js
  Release   → GET /api/release/* → server/release/

1.3 Abgrenzung zu Pauli BestPrice Thailand
----------------------------------
  | Aspekt              | OSG Homepage                    | Pauli BestPrice Thailand |
  |---------------------|---------------------------------|-------------------------|
  | Frontend            | Viele HTML + js/*.js            | Eine index.html SPA     |
  | Server-Einstieg     | server/index.js                 | 02_Quellcode/Core_Logik/server.js |
  | Preis-/Shop-Logik   | Nur OmniQR (Zahlungs-QR)        | Vergleich, Affiliate, VIP |
  | Avatar / Live-Chat  | Marken-Vorlesen (TTS)           | Pauli Live-Konversation |
  | i18n                | assets/locales/*.json (i18next)| i18n-locales.js (T-Objekt)|

2. SERVER-STARTPUNKT (IST-ZUSTAND)
================================================================================

2.1 Primärer Einstieg — NICHT im Repo-Root
------------------------------------------
  Datei:     server/index.js
  Package:   server/package.json  →  "start": "node index.js"
  Deploy:    render.yaml → rootDir: server, startCommand: npm start
  Port:      process.env.PORT || LOCAL_WEB_PORT || 8080

2.2 SITE_ROOT-Auflösung
-----------------------
  const SITE_ROOT = path.join(__dirname, '..');
  → Statische Dateien (HTML, css/, js/, assets/) werden aus dem Repo-Root
    ausgeliefert, während die Server-Logik in server/ liegt.

  Bewertung: [TEILWEISE strukturiert]
  • Server ist bereits aus dem Root getrennt (besser als Pauli vor Welle 1).
  • package.json liegt nur unter server/, nicht im Repo-Root.
  • render.yaml rootDir: server erzwingt korrekten Startpfad auf Render.

2.3 Registrierte API-Routen (server/index.js)
---------------------------------------------
  GET  /health
  POST /api/speak, /api/speak/stream     — Marken-TTS (tts-router)
  USE  /api                                — release-Router
  USE  /api/ops                            — Ops/Diagnostik
  USE  /api/i18n                           — Locale-Merge/Build
  USE  /omniqr-ai-for-tourist-of-thailand/api/*  — Zahlungs-QR-Modul
